React 中使用 Ref

发表于2023-07-31|更新于2023-07-31
|阅读量:

一、字符串

  1. 在 Dom 元素上使用，通过 this.refs[name] 进行引用
  2. 在类组件上使用，也是通过 this.refs[name] 来进行引用

二、回调函数

  • 回调函数就是在dom节点或组件上挂载函数，函数的入参是dom节点或组件实例，达到的效果与字符串形式是一样的，都是获取其引用
  • 回调函数的触发时机
    1. 组件渲染后，即componentDidMount后
    2. 组件卸载后，即componentWillMount后，此时，入参为null
    3. ref改变后
  1. 在 Dom 节点上使用回调函数
  2. 在类组件上使用
  3. 可用通过props跨级传递的方式来获取子级 Dom 节点或组件实例

三、React.createRef

  • 在类组件中，使用此方法来创建ref。将其赋值给一个变量，通过ref挂载在dom节点或组件上，该 ref 的 current 属性将能拿到dom节点或组件的实例

四、useRef

  • 在函数式组件中，使用此方法来创建ref。将其赋值给一个变量，通过ref挂载在dom节点或组件上，该 ref 的 current 属性将能拿到dom节点或组件的实例

五、注意

  1. 避免滥用：Ref 是一种直接操作 DOM 的方法，在大多数情况下，我们应该优先考虑使用 React 的数据流和状态管理来处理组件逻辑
  2. 避免在 render 中使用 Ref：render 方法可能会在每次渲染时被调用，频繁地创建 Ref 可能会导致性能问题
  3. 与 React 的生命周期和钩子函数结合使用：对于类组件，Ref 可以结合 componentDidMount、componentDidUpdate 等生命周期函数使用。对于函数组件，Ref 可以结合 useEffect 钩子使用

六、总结

Ref 是 React 中非常有用的特性，它允许我们直接访问 DOM 元素或组件实例，处理一些常见的场景。然而，使用 Ref 时需要谨慎，避免滥用，同时结合 React 的生命周期和钩子函数来合理地应用
